# Closely AI - Local Development Setup

This guide details steps to clone, configure, and boot the Closely AI application on a local machine for testing.

---

## 1. System Prerequisites
- **Python**: v3.11+
- **Node.js**: v18+ (with npm)
- **Docker**: Docker Desktop (required to run PostgreSQL + pgvector and Redis locally)

---

## 2. Step-by-Step Installation

### Step A: Clone & Setup Environment
1. Copy the environment template:
   ```bash
   cp .env.example .env
   ```
2. Open `.env` and fill in local credentials (see `38_environment_variables.md` for requirements).

### Step B: Boot Services in Docker
1. Spawn the database and cache in the background:
   ```bash
   docker-compose up -d db redis
   ```
2. Verify Postgres and Redis containers are healthy.

### Step C: Backend Setup
1. Create a virtual environment:
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  # On Windows: .\venv\Scripts\activate
   ```
2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run Alembic migrations to generate database schemas:
   ```bash
   alembic upgrade head
   ```
4. Start the FastAPI development server:
   ```bash
   uvicorn app.main:app --reload --port 8000
   ```

### Step D: Frontend Setup
1. Open a new terminal window:
   ```bash
   cd frontend
   npm install
   ```
2. Launch the Vite development server:
   ```bash
   npm run dev
   ```
3. Access the dashboard at `http://localhost:5173`.

---

## 3. Webhook Integration Testing via Ngrok
Meta Cloud API webhooks require a public HTTPS URL. To route Meta requests to your local computer:
1. Start ngrok:
   ```bash
   ngrok http 8000
   ```
2. Copy the secure HTTPS URL generated by ngrok (e.g. `https://abcd.ngrok-free.app`).
3. Set your Meta App webhook callback URL to: `https://abcd.ngrok-free.app/api/webhooks/whatsapp`.
4. Enter the matching verification token configured in your `.env`.
